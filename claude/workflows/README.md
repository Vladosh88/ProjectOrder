# Готовые Workflow для Claude Code

**От Никиты Велса** — бонус к ролику про Dynamic Workflows.

Шесть готовых воркфлоу для Claude Code. Каждый — это скрипт, который поднимает кучу агентов, гонит задачу «веером» и отдаёт один готовый результат. Менять код не нужно — просто положи файлы к себе и запусти.

> 💡 **Заодно это образец промпт-инжиниринга.** Промпты внутри каждого `agent()` написаны по структуре **Роль → Контекст → Задача → Ограничения → Формат** — открой любой файл и подсмотри, как формулировать запросы, чтобы агенты не галлюцинировали и отдавали ровно то, что нужно.

## Что в архиве

| Файл | Команда | Что делает |
|------|---------|-----------|
| `analyze-competitors.js` | `/analyze-competitors` | Анализ конкурентов: по агенту на конкурента → сводная таблица + пробелы рынка |
| `deep-research.js` | `/deep-research` | Глубокий ресёрч темы: углы → поиск → проверка фактов (3 голоса) → отчёт со ссылками |
| `data-analytics.js` | `/data-analytics` | Аналитика массива данных (отзывы, заявки, строки): темы, тональность, инсайты |
| `content-angles.js` | `/content-angles` | Контент-идеи под тему: углы из разных линз → судейская панель → топ-3 |
| `audience-insights.js` | `/audience-insights` | Анализ аудитории из кастдевов/комментариев/анкет: боли, возражения, сегменты, цитаты |
| `codebase-audit.js` | `/codebase-audit` | Аудит кода: поиск проблем веером → состязательная проверка → только подтверждённое |

---

## Что нужно

- **Claude Code v2.1.154 или новее** (Dynamic Workflows вышли в research preview).
- **Тариф:** на Max / Team / API воркфлоу включены по умолчанию. На Pro — включи тумблер в `/config` (строка про Dynamic workflows). В организациях по умолчанию выключено — попроси админа включить.
- Для `deep-research` нужен доступ к веб-поиску (WebSearch).

> ⚠️ Воркфлоу поднимают много агентов и **тратят заметно больше токенов**, чем обычный чат. Всё считается в твой лимит. Начинай с маленькой задачи, чтобы прикинуть расход.

---

## Шаг 1. Импорт — куда положить файлы

Положи `.js`-файлы в одну из папок:

- **Только для одного проекта** (расшарить с командой через репозиторий):
  ```
  <твой-проект>/.claude/workflows/
  ```
- **Глобально, для всех проектов** (только для тебя):
  ```
  ~/.claude/workflows/
  ```

Например:

```bash
# глобально
mkdir -p ~/.claude/workflows
cp *.js ~/.claude/workflows/
```

Как только файл лежит в этой папке — он сразу доступен как команда `/<имя>` (имя берётся из поля `name` внутри файла, а не из имени файла).

Проверить, что подхватилось:

```
/workflows
```

---

## Шаг 2. Запуск

Просто вызови команду и опиши задачу — Claude сам передаст данные внутрь скрипта:

```
/analyze-competitors разбери конкурентов Cursor, GitHub Copilot, Windsurf в нише AI-редакторов кода
```

```
/deep-research куда пойдёт курс рубля к доллару в ближайший квартал
```

```
/content-angles придумай идеи для ролика на тему «AI-агенты для малого бизнеса»
```

При первом запуске Claude Code покажет, что собирается сделать, и спросит подтверждение. Дальше следи за прогрессом в:

```
/workflows
```

Там видно фазы, число агентов, потраченные токены и время. Любой запуск можно **остановить** прямо оттуда — уже сделанная работа не пропадёт.

### Как передать данные точнее (необязательно)

Каждый воркфлоу понимает структурированный ввод. Если хочешь задать его явно, попроси Claude передать `args`:

| Воркфлоу | Пример `args` |
|----------|---------------|
| analyze-competitors | `{ "niche": "онлайн-курсы по AI", "competitors": ["A","B","C"] }` |
| deep-research | `"Твой вопрос целиком"` |
| data-analytics | `{ "goal": "выдели боли клиентов", "items": ["отзыв1","отзыв2"] }` |
| content-angles | `{ "topic": "AI для бизнеса", "format": "YouTube-ролик" }` |
| audience-insights | `{ "sources": ["кастдев...","комментарии...","анкета..."] }` |
| codebase-audit | `{ "paths": ["src/api/","src/auth.ts"], "focus": "security" }` |

## Контроль расхода (уже встроено)

Эти воркфлоу написаны так, чтобы не жечь лишнего:

- **Дешёвое — дешёвой моделью.** Сбор и разметка идут на быстрой модели (`haiku`/`sonnet`), а умная и дорогая (`opus`) подключается только на финальную сборку.
- **Ограничители на объём.** Там, где данных может быть много (`data-analytics`, `audience-insights`, `codebase-audit`), стоит потолок на число кусков/файлов — цикл не уйдёт в бесконечность.
- **Свой бюджет.** Можешь задать лимит прямо в запросе, например: «используй не больше 10k токенов».
- **Система держит сама.** Одновременно работает максимум 16 агентов, всего за запуск — не больше 1000.

---

## Как поправить под себя

Файлы — обычный JavaScript, читаются почти как текст. Можно открыть и поменять:

- список линз/углов (в `content-angles.js` — массив `LENSES`),
- модели по фазам (`model: 'haiku' | 'sonnet' | 'opus'`),
- потолки на объём (`MAX`, `CHUNK`, `MAX_CHUNKS`).

Или просто попроси Claude: «открой `content-angles.js` и добавь линзу про FOMO» — он отредактирует сам.

## Если что-то не так

- **Команды не видно в `/workflows`** → проверь версию Claude Code (`v2.1.154+`) и что файл лежит в `~/.claude/workflows/` или `<проект>/.claude/workflows/`.
- **На Pro ничего не запускается** → включи Dynamic workflows в `/config`.
- **`deep-research` не ищет** → нужен доступ к веб-поиску (WebSearch).

Это research preview — поведение и лимиты могут меняться между версиями.
